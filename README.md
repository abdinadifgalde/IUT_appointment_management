# IUT Appointment System

A Flask-based university appointment booking system for IUT Dhaka.

---

## 🚀 Deploy to Render (Recommended)

### One-click setup
1. Push this repo to GitHub.
2. Go to [render.com](https://render.com) → **New** → **Blueprint** → connect your repo.
   - Render will auto-detect `render.yaml` and configure everything.
3. Set the two required environment variables when prompted:
   - `MAIL_USERNAME` — your Gmail address
   - `MAIL_PASSWORD` — your [Gmail App Password](https://myaccount.google.com/apppasswords)
4. Click **Apply** and wait ~2 minutes for the first deploy.

> `SECRET_KEY` is auto-generated by Render. The SQLite database is stored on a persistent 1 GB disk so data survives redeploys.

### Manual setup (if not using Blueprint)
1. **New Web Service** → connect repo
2. **Build Command:** `pip install -r requirements.txt`
3. **Start Command:** `gunicorn --worker-class eventlet -w 1 --bind 0.0.0.0:$PORT app:app`
4. **Environment Variables:**

   | Key | Value |
   |-----|-------|
   | `SECRET_KEY` | any long random string |
   | `FLASK_ENV` | `production` |
   | `MAIL_USERNAME` | your Gmail |
   | `MAIL_PASSWORD` | your Gmail App Password |

5. **Disk** → Mount Path: `/opt/render/project/src/database`, Size: 1 GB

## 💻 Local Development

```bash
pip install -r requirements.txt
python app.py
```

Visit `http://localhost:5000`

### Seed test data
```bash
python test_app.py
```
Creates: Admin `admin@iut-dhaka.edu / admin123` and Student `abdinadiif@iut-dhaka.edu / student123`

### Default super admin (auto-created on first run)
- Email: `superadmin@iut-dhaka.edu`  
- Password: `SuperAdmin@2026!`

## 🗄️ Database Migrations

```bash
flask db init && flask db migrate -m "initial" && flask db upgrade
# After any model change:
flask db migrate -m "describe change" && flask db upgrade
```

## ✨ Features
- IUT email enforcement, role-based access (Super Admin / Admin / Officer / Student)
- QR code appointment slips, live status via Socket.IO
- Email notifications, waitlist auto-promotion, analytics dashboard
- Dark mode, session timeout, rate limiting, audit logs
